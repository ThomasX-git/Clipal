## v0.11.1

### English

#### Highlights

- Refined Web UI feedback messaging so success and error states are more specific, easier to scan, and better matched to the part of the interface where the action happened.
- Moved `CLI Takeover` success feedback into the relevant card instead of reusing a page-wide banner, while upgrading the shared global alert into a proper toast component.

#### What Changed

- Improved: the shared Web UI alert now renders as a top-right toast with clearer title/body structure, stacking behavior, and dismiss controls
- Improved: `CLI Takeover` apply / restore results now stay inside the affected card with embedded success or error feedback instead of relying on a global success banner
- Improved: provider page actions now emit more contextual feedback, including the affected client or provider, instead of collapsing multiple flows into the same generic “saved successfully” message
- Fixed: takeover success guidance about restarting the client or opening a new session now remains attached to the relevant integration card, so the next-step instruction is easier to keep in view

### 中文

#### 亮点

- 继续打磨 Web UI 的反馈消息，让成功和错误提示更具体、更易扫读，也更符合操作发生的位置。
- `CLI Takeover` 的成功反馈不再复用全局横幅，而是回到对应卡片内部；同时，全局 alert 也升级成了真正的 toast 组件。

#### 变更内容

- 改进：共享的 Web UI alert 现在以右上角 toast 呈现，具备更清晰的标题/正文结构、堆叠行为和关闭能力
- 改进：`CLI Takeover` 的 apply / restore 结果现在直接显示在对应卡片内部，使用嵌入式 success / error 反馈，而不再依赖全局成功提示
- 改进：Providers 页面中的各类操作反馈现在会带上受影响的 client 或 provider，不再把多种流程压成同一句泛化的“保存成功”
- 修复：takeover 成功后关于“重启 client 或新开 session”的后续提示，现在会留在对应 integration 卡片里，更容易让用户保持预期
