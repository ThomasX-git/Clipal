## v0.5.4

### Highlights (EN)

- Improved: provider failover logs now use clearer user-facing wording instead of transport-oriented tuples.
- Improved: `last_switch` now records hop-by-hop provider switches, avoiding misleading "first provider -> final provider" summaries in multi-hop failover.
- Improved: `last_request` now covers early terminal outcomes such as incomplete responses, all providers unavailable, and local request-setup failures.
- Improved: Web UI status now shows normalized labels/details for provider state, last switch, and last request.
- Improved: macOS notifications use a bundled app icon and cleaner provider-switch summaries.

### 亮点 (中文)

- 改进：provider failover 日志改为更清晰的用户语义表达，不再直接输出偏传输层的内部 tuple。
- 改进：`last_switch` 现在按逐跳记录 provider 切换，避免多跳 failover 时出现“首个 provider -> 最终 provider”的误导性总结。
- 改进：`last_request` 现在覆盖更多提前终止场景，包括响应不完整、全部 provider 不可用，以及本地请求构造失败等终态。
- 改进：Web 状态页现在使用归一化的标签与说明展示 provider 状态、最近一次切换和最近一次请求结果。
- 改进：macOS 通知改用内置应用图标，并优化 provider 切换通知文案。
