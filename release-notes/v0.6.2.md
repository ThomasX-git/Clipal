## v0.6.2

### English

#### Highlights

- Normalized runtime, Web, and config-management boundaries across the codebase.
- Made Web configuration writes safer by serializing mutations and rejecting false-success reloads.
- Tightened provider reload behavior so harmless edits preserve runtime state, while materially changed upstreams do not inherit stale cooldown or circuit-breaker state.
- Improved service status consistency between the CLI and Web UI.
- Expanded direct regression coverage across app lifecycle, Web config flows, proxy reloads, service status, self-update, and CLI status paths.

#### What Changed

- Improved: extracted application bootstrap and shutdown orchestration into `internal/app`
- Improved: extracted Web config persistence/reload flow into `internal/web/config_store.go`
- Improved: unified Web JSON error handling in `internal/web/errors.go`
- Improved: restored lifecycle and shutdown events to the configured logger sink
- Improved: aligned `/api/service/status` with parsed service state instead of command-success heuristics
- Improved: strengthened config validation, including duplicate provider-name rejection per client
- Improved: restored the default log retention behavior to 7 days
- Improved: stabilized YAML formatting and round-trip behavior for Web-managed config files
- Improved: pinned CI/security workflow dependencies and added contributor/lint tooling
- Fixed: Linux lint issues and CI smoke-test timing/logging robustness

### 中文

#### 亮点

- 对运行时、Web 管理接口和配置管理边界做了一次整体规范化整理。
- Web 配置写入链路更安全：串行化配置变更，并避免“写盘成功但 reload 未生效”的假成功。
- provider reload 行为更稳：无害配置变更会保留运行时状态，但实质性 upstream 变化不会继承旧的 cooldown 或熔断状态。
- CLI 和 Web UI 的 service status 语义保持一致。
- 为应用生命周期、Web 配置、proxy reload、service status、自更新和 CLI status 补齐了大批直接回归测试。

#### 变更内容

- 改进：将应用启动与关闭编排收敛到 `internal/app`
- 改进：将 Web 配置读写与 reload 流程抽到 `internal/web/config_store.go`
- 改进：统一 Web JSON 错误输出到 `internal/web/errors.go`
- 改进：生命周期和 shutdown 日志重新纳入配置 logger sink
- 改进：`/api/service/status` 改为基于解析后的服务状态，而不是命令是否执行成功
- 改进：增强配置校验，包括同一 client 下重复 provider 名称直接拒绝
- 改进：默认日志保留行为恢复为 7 天
- 改进：提升 Web 管理配置文件的 YAML 格式稳定性与 round-trip 行为
- 改进：补齐 CI / 安全扫描依赖固定，以及 contributor / lint 工具链
- 修复：补齐 Linux lint 问题，并增强 CI smoke 测试等待与失败日志输出
