## v0.11.2

### English

#### Highlights

- Smoothed out the CLI flow around service restarts so common interactive usage is handled more directly and mistakes get clearer guidance.
- Improved post-update guidance so users running Clipal as a background service immediately see the next restart step after updating.

#### What Changed

- Added: `clipal restart` as a shortcut for `clipal service restart`
- Improved: root CLI help now shows top-level commands and examples instead of only raw startup flags
- Improved: `clipal service --help` now prints service usage directly instead of failing with a missing-action error
- Fixed: unknown top-level commands no longer fall through to the default server start path
- Fixed: mistyped commands such as `clipal restart-now` now fail fast with usage guidance instead of accidentally trying to start another server instance
- Improved: `clipal update` now prints a restart hint for users who run Clipal as a background service
- Improved: docs now explicitly show `clipal restart` as the interactive shortcut while keeping `clipal service restart` as the canonical scripted form

### 中文

#### 亮点

- 优化了与后台服务重启相关的 CLI 交互路径，让常见的手工输入更顺手，误用时也能拿到更明确的提示。
- `clipal update` 完成后会直接提示后台服务用户下一步该执行重启，减少升级后的猜测成本。

#### 变更内容

- 新增：`clipal restart`，作为 `clipal service restart` 的快捷写法
- 改进：根命令帮助现在会展示一级命令和示例，而不再只列出启动参数
- 改进：`clipal service --help` 现在会直接输出 service 用法，不再因为缺少 action 报错
- 修复：未知一级命令不再落回默认启动服务路径
- 修复：像 `clipal restart-now` 这样的误输入现在会快速失败并给出 usage，而不是意外尝试再启动一个服务实例
- 改进：`clipal update` 现在会为后台服务用户补充重启提示
- 改进：文档中明确展示 `clipal restart` 作为交互式快捷命令，同时保留 `clipal service restart` 作为脚本和文档中的规范写法
