## v0.17.3

### English

#### Highlights

- Restored Claude OAuth request attribution for current Claude Code traffic.
- Aligned Codex OAuth response handling with the latest local Codex OAuth behavior.
- Synced the Gemini OAuth Code Assist wrapper with the current official Gemini CLI package.

#### What Changed

- Fixed: Claude OAuth requests now preserve and sanitize the current attribution and billing metadata expected by Claude Code.
- Fixed: Codex OAuth requests now use the updated Responses wrapper shape, response rewrite path, retry behavior, and provider override cache handling.
- Fixed: Gemini OAuth requests now match current Code Assist headers, `generateContent`, `streamGenerateContent`, and `countTokens` request envelopes.
- Improved: Gemini OAuth setup now supports official Google Cloud project environment variables, rejects numeric project numbers, and returns clearer account validation errors.
- Improved: Gemini OAuth response rewriting now tolerates upstream responses that omit `Content-Type`, and Google RPC quota errors are classified more accurately.

#### Thanks

- Thanks to the issue reporters and live OAuth verification that caught the Claude, Codex, and Gemini upstream compatibility drift before a broader release.
